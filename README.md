## Phân tích layout

1. Phân tích các thành phần cơ bản (Bộ khung)
   - Header (Đầu trang): Chứa logo, thanh menu điều hướng chính và đôi khi là thanh tìm kiếm.
   - Navigation Bar (Thanh điều hướng): Giúp người dùng di chuyển giữa các trang con một cách dễ dàng.
   - Main Content (Nội dung chính): Khu vực quan trọng nhất, hiển thị thông tin mà người dùng đang tìm kiếm (văn bản, hình ảnh, video).
   - Sidebar (Thanh bên): Thường chứa các thông tin bổ trợ như danh mục, bài viết liên quan, quảng cáo hoặc widget.
   - Footer (Chân trang): Chứa thông tin liên hệ, bản quyền, liên kết mạng xã hội và các chính sách.
2. Các phần tử chung.
   - Xác định tất cả có bố cục html css giống nhau
3. Phân tích nên chọn bố cục layout nào phù hợp(grid/flex)
4. Responsive
   - Phân tích xem layout có tự động thay đổi để hiển thị tốt trên cả máy tính, máy tính bảng và điện thoại hay không.
   - mobile - tablet nhỏ: 1 cột
   - tablet lớn - laptop: 2 cột
   - above : 3-4 cột

## Obsidian

1. Cài đặt
2. Formmat:
   - YAML Frontmatter — luôn mở đầu với --- để gắn metadata (title, date, tags, status)
   - Tiêu đề phân cấp — 1 H1 duy nhất, H2 cho mục lớn, H3 cho mục nhỏ, không nhảy cấp
   - Wiki-link — dùng [[tên-file]] để liên kết giữa các ghi chú, tạo mạng lưới kiến thức
   - Tags phân cấp — ví dụ #học-tập/toán thay vì chỉ #toán
   - Callout blocks — > [!note], > [!tip], > [!warning] để làm nổi bật ý quan trọng
   - Mục lục tự động — đặt ngay sau H1 với plugin Dynamic TOC
   - Tên file chuẩn — dùng prefix như MOC - hay YYYY-MM-DD, tránh ký tự đặc biệt
   - Section "Xem thêm" — cuối file ghi nguồn tham khảo và link liên quan
3. Cài agent
